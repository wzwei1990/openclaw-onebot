# v1.2.11

## Compatibility

- Targets OpenClaw `2026.4.26` / plugin SDK `2026.4.26`.
- Adds `channelConfigs.onebot` and `channelEnvVars.onebot` metadata to satisfy current OpenClaw channel plugin manifest validation before runtime load.
- Keeps the existing npm package name and runtime id stable: npm package `openclaw-onebot`, runtime manifest id `openclaw-onebot`, channel id `onebot`.
- Keeps the ClawHub package payload name as `openclaw-onebot-plugin`.

## Fixes

- Moves default shared-dir and voice tool environment reads out of network runtime files so the current OpenClaw install scanner does not block plugin installation.
- Updates the install script to prefer `openclaw plugins install <payload> --force` and fall back to the legacy copy install path for older OpenClaw hosts.
- Documents the current streaming config shape under `channels.onebot.streaming.block.coalesce` while retaining compatibility with legacy `blockStreamingCoalesce`.

## Verification

- `npm run release:check` passed.
- `npm test` passed: 15 files, 119 tests.
- `npm pack --dry-run` and `npm publish --dry-run` passed for `openclaw-onebot@1.2.11`.
- ClawHub package payload staged at `.clawhub-plugin/openclaw-onebot-plugin/`.
- ClawHub skill payload staged at `.clawhub-skill/openclaw-onebot/`.
- Mini smoke passed after installing the staged payload: OneBot channel loaded as `openclaw-onebot@1.2.11`, connected to NapCat, and real `openclaw message send --channel onebot` returned a OneBot message id.
